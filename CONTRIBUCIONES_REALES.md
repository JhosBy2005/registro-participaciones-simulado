# Guia para registrar participacion real en GitHub

Para que GitHub muestre commits reales de cada integrante, cada persona debe trabajar desde su propia cuenta.

## Pasos

1. El dueno del repositorio invita a cada integrante desde `Settings > Collaborators`.
2. Cada integrante acepta la invitacion en GitHub.
3. Cada integrante clona el repositorio o lo abre desde GitHub Desktop.
4. Cada integrante realiza un cambio real y especifico.
5. Cada integrante hace commit y push desde su propia cuenta.

## Ejemplos de mensajes de commit

```text
feat: improve semantic structure for main sections
style: refine responsive spacing for final layout
fix: improve form validation feedback
docs: update fraud warning content
chore: prepare final delivery files
```

## Nota

No uses autores, fechas o cuentas de otras personas para simular actividad. Si varias personas colaboran realmente en un mismo cambio, pueden usar `Co-authored-by` en el mensaje del commit.
